Flask CRUD — One-Page Skeleton

Memorize the shape, look up the details. Every CRUD app is this, repeated per resource.

The mental model

OpRouteMethod(s)Core action
Create/newGET + POSTdb.session.add(obj)
Read/GETModel.query.all()
Update/<id>/editGET + POSTmutate fields + commit
Delete/<id>/deletePOSTdb.session.delete(obj)

Pattern: GET renders a form/list → POST mutates then redirect(url_for(...)). Always commit(). Always get_or_404().

__init__.py — factory + db

from flask import Flask
from flask_sqlalchemy import SQLAlchemy

db = SQLAlchemy()

def create_app():
    app = Flask(__name__)
    app.config["SQLALCHEMY_DATABASE_URI"] = \
        "sqlite:///app.db"   # or postgres, see below
    db.init_app(app)
    from . import routes
    app.register_blueprint(routes.bp)
    with app.app_context():
        db.create_all()
    return app

models.py

from . import db

class Item(db.Model):
    id   = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String, nullable=False)

routes.py — the 4 endpoints

from flask import (Blueprint, render_template,
                   request, redirect, url_for)
from . import db
from .models import Item

bp = Blueprint("items", __name__)

@bp.route("/")                              # READ
def index():
    return render_template("list.html",
                           items=Item.query.all())

@bp.route("/new", methods=["GET", "POST"])  # CREATE
def create():
    if request.method == "POST":
        db.session.add(Item(name=request.form["name"]))
        db.session.commit()
        return redirect(url_for("items.index"))
    return render_template("form.html", item=None)

@bp.route("/<int:id>/edit",
          methods=["GET", "POST"])          # UPDATE
def edit(id):
    item = Item.query.get_or_404(id)
    if request.method == "POST":
        item.name = request.form["name"]
        db.session.commit()
        return redirect(url_for("items.index"))
    return render_template("form.html", item=item)

@bp.route("/<int:id>/delete",
          methods=["POST"])                 # DELETE
def delete(id):
    item = Item.query.get_or_404(id)
    db.session.delete(item)
    db.session.commit()
    return redirect(url_for("items.index"))

Talk-track (say these out loud — they're what's graded)

  • Validation: check required fields, flash an error, re-render the form instead of crashing.
  • Security: ORM/parameterized queries (never f-string SQL); hash passwords with generate_password_hash if auth comes up.
  • Not-found: get_or_404() rather than letting a missing id 500.
  • Migrations: "create_all() for dev; I'd use Alembic / Flask-Migrate in production."
  • REST variant: return jsonify(...) + status codes (201 create, 204 delete) instead of redirects if they want an API.
  • Ask first: "Docs OK? AI assistant allowed? Is a DB provided or should I set one up?"
